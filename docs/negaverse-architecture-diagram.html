negaverse — negative dataset generation pipeline

negaverse takes a dataset of known positive interactions and generates better negative examples for machine learning. Instead of using random pairs, it creates candidates, removes known positives, scores the remaining pairs, and exports separate training and evaluation datasets with confidence scores and provenance.

The main idea is simple: start broad, filter carefully, score transparently, and export two datasets for different purposes. The training set contains harder examples for learning. The evaluation set is better matched to the positives, so it is more useful for fair benchmarking. Every generated pair keeps its score details and provenance, so users can understand why it was included.
